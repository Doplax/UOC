Una vez realizada una aplicación para web y móvil, realizaremos una ampliación que afectará a ambos desarrollos, desarrollaremos la posibilidad de recibir notificaciones push haciendo uso de dos herramientas de Firebase: Cloud Firebase Messaging y Cloud Functions.

Una de las características más usuales hoy en día con este tipo de aplicaciones nos va a permitir visualizar cambios en la base de datos

Comenzaremos con la aplicación desarrollada con React Native y para ello usaremos react native firebase (https://rnfirebase.io/), en concreto cloud messaging, por lo que seguiremos las instrucciones del proveedor (https://rnfirebase.io/messaging/usage).
A través de la consola de Firebase crearemos una nueva aplicación o proyecto para Android

Registraremos nuestro dispositivo mediante el json que nos proporciona el proveedor

Añadiremos los permisos a nuestra aplicación y añadiremos el código necesario dentro de app.js para poder recibir el mensaje desde Firebase. Podemos probar desde nuestro proyecto generado la correcta recepción de la notificación
FP067_Producto4_IMG03.PNG

Toca el turno de programar dentro de nuestra base de datos un envío de mensaje usando las notificaciones de firebase cuando se produzca una escritura o modificación de las bases de datos. Para ello usaremos cloud functions.
Preparar el entorno para la programación del envío de mensajes, siguiendo las instrucciones del fabricante, https://firebase.google.com/docs/functions/get-started#initialize-your-project
Importar los módulos para comenzar a programar nuestro trigger siguiendo las instrucciones del fabricante , https://firebase.google.com/docs/functions/get-started#import-the-required-modules-and-initialize-an-app
Activar el envío de un push messaging usando los eventos onWrite() y onUpdate() de la base de datos (https://firebase.google.com/docs/functions/database-events#set_the_event_handler)
Realizar todo el proceso anterior descrito, para Angular usando Firebase Messaging y un proyecto creado para nuestra aplicación con Angular
